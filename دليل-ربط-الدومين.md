## 🚀 طريقتان لربط الدومين

### الطريقة الأولى: استخدام Vercel Nameservers (الأسهل) ⭐

> **موصى بها للمبتدئين - Vercel يدير كل شيء تلقائياً!**

#### الخطوات:

1️⃣ **في Vercel Dashboard:**
   - اذهب إلى: https://vercel.com/dashboard
   - اختر مشروع: `delivery-form-mshari`
   - Settings → Domains → Add Domain
   - أضف: `firstlineon.com`

2️⃣ **في لوحة تحكم مزود النطاق:**
   - ابحث عن إعدادات Nameservers
   - غيّرها إلى:
   ```
   ns1.vercel-dns.com
   ns2.vercel-dns.com
   ```

3️⃣ **انتظر:**
   - 24-48 ساعة لانتشار Nameservers
   - Vercel سيتولى باقي الإعدادات تلقائياً

#### ✅ مميزات هذه الطريقة:
- إعداد أبسط وأسهل
- إدارة DNS كاملة من Vercel
- شهادة SSL تلقائية
- دعم لجميع النطاقات الفرعية

---

### الطريقة الثانية: استخدام A/CNAME Records (للمحترفين)

> **إذا كنت تريد الاحتفاظ بإعدادات DNS الحالية**

#### الخطوات:

1️⃣ **في Vercel Dashboard:**
   - نفس الخطوات السابقة
   - أضف: `firstlineon.com` و `www.firstlineon.com`

2️⃣ **في لوحة تحكم مزود النطاق:**
   
   **A Record (للدومين الرئيسي):**
   ```
   النوع: A
   الاسم: @ (أو اتركه فارغاً)
   القيمة: 76.76.21.21
   TTL: تلقائي
   ```

   **CNAME Record (لـ www):**
   ```
   النوع: CNAME
   الاسم: www
   القيمة: cname.vercel-dns.com
   TTL: تلقائي
   ```

3️⃣ **انتظر:**
   - 5-60 دقيقة فقط!

#### ⚠️ ملاحظات مهمة:
- احذف أي سجلات A أو AAAA قديمة
- تأكد من عدم وجود تعارض في السجلات

## ❓ حل المشاكل الشائعة

### المشكلة: "Invalid Configuration"
**الحلول:**
1. تأكد من كتابة DNS records بدون https://
2. احذف المسافات الزائدة
3. تأكد من استخدام @ وليس firstlineon.com في A record

### المشكلة: الموقع لا يفتح
**الحلول:**
1. انتظر وقتاً إضافياً (الانتشار يحتاج وقت)
2. امسح DNS cache:
   ```cmd
   ipconfig /flushdns
   ```
3. جرب متصفح آخر أو وضع التصفح الخفي

### المشكلة: شهادة SSL غير آمنة
**الحل:** انتظر 15 دقيقة - Vercel ينشئ الشهادة تلقائياً
